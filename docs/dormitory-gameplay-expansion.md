# Dormitory Gameplay Expansion

## 1. 目标
- 让寝室系统从“单点互动”升级为“可长期游玩的养成玩法”。
- 强化三个体验支点：决策压力、长期成长、随机惊喜。
- 为后续迭代提供统一设计基线，减少临时改需求导致的返工。

## 2. 核心扩展玩法清单

### 2.1 时间管理回合（早/午/晚）
- 每天分为 3 个时段，每个行动消耗时段。
- 玩家需在“刷好感 / 升设施 / 做事件”之间取舍。
- 价值：把当前互动系统变成策略循环，提升可玩时长。

### 2.2 角色需求系统（当日心愿）
- 每个 CHAR 每天刷新 1-2 个偏好需求（例如：想聊天、想训练、想独处）。
- 满足需求触发高收益或特殊事件；忽略会降低心情或触发负面状态。
- 价值：角色更“活”，玩家决策更有方向。

### 2.3 场景事件链（多步分支）
- 事件由单次结算扩展为 2-4 步剧情链。
- 前一步选择影响后续节点和最终奖励。
- 价值：形成“剧情记忆点”和可复玩分支。

### 2.4 关系阶段与专属剧情
- 关系到达阈值解锁阶段（陌生 -> 熟悉 -> 亲密 -> 羁绊）。
- 每阶段开放专属场景、动作、剧情、立绘或语音触发条件。
- 价值：长期目标明确，玩家有持续推进动力。

### 2.5 双角色联动玩法
- 允许双人互动，按角色相性计算加成/冲突。
- 可触发“协同事件”或“三人关系张力事件”。
- 价值：组合策略丰富，内容增长速度高。

### 2.6 家具与功能位系统（Build）
- 场景不仅升级等级，还可切换模块（效率流/恢复流/社交流等）。
- 不同模块改变互动收益曲线与事件权重。
- 价值：让“场景养成”有流派差异而非纯数值堆叠。

### 2.7 周目标与赛季主题
- 每周设置主题（考试周、停电周、节日周）。
- 全局事件池和奖励规则按主题切换。
- 价值：维持新鲜感，降低内容消耗感。

### 2.8 轻量小游戏接入
- 特定事件或场景触发 30-60 秒小游戏（节奏/反应/拼图）。
- 结果映射到奖励倍率、事件成功率或隐藏剧情权重。
- 价值：提升操作参与感，打破纯文本疲劳。

### 2.9 回忆相册与图鉴收集
- 记录关键事件结局、关系阶段节点、场景成就。
- 提供收集进度和缺失提示。
- 价值：增强留存与长期完成欲。

### 2.10 角色记忆与偏好学习
- 系统记录玩家习惯选择与互动风格。
- 角色后续会基于“记忆标签”改变对话和事件反应。
- 价值：增强角色个性真实感，提高沉浸感。

## 3. 推荐开发优先级

### Phase A（先做，性价比最高）
- 时间管理回合
- 角色需求系统
- 场景事件链

### Phase B（中期，增强深度）
- 关系阶段与专属剧情
- 家具与功能位系统
- 双角色联动玩法

### Phase C（后期，拉长生命周期）
- 周目标与赛季主题
- 回忆相册与图鉴收集
- 角色记忆与偏好学习
- 轻量小游戏接入

## 5. 玩法循环示例（单日）
1. 刷新当日心愿与主题修正。
2. 玩家在 3 个时段内执行行动。
3. 行动推进关系、场景、事件链、收集进度。
4. 夜间结算：心愿完成、关系变化、次日状态生成。

## 6. 成功指标建议
- 单次游玩时长（平均）
- 7 日留存 / 14 日留存
- 日均互动次数（chat/gift/event/scene）
- 事件链完成率
- 关系阶段推进率
- 图鉴完成率

## 7. 风险与控制
- 内容膨胀：优先做“可复用规则系统”，再堆文本内容。
- 数值失衡：统一通过收益公式和倍率表集中调参。
- 保存兼容：每次新增字段需保持默认值与向下兼容逻辑。

## 8. 后续可直接执行的下一步
- 先在现有寝室系统落地 Phase A 的最小闭环：
  - 回合时间槽
  - 当日心愿
  - 2 条多步事件链
- 完成后再扩展 Phase B，避免一次性开发过重。
